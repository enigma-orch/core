# enigma-infra

Shared infrastructure for the enigma stack. Owns the local Postgres,
object store, cache, optional graph DB, **and the database schema** that
both services consume.

## Services

| Service  | Image                     | Port(s)      | Purpose                                |
|----------|---------------------------|--------------|----------------------------------------|
| postgres | pgvector/pgvector:pg16    | 5432         | Primary relational DB + pgvector       |
| migrate  | ghcr.io/amacneil/dbmate:2 | -            | Applies migrations from `db/migrations`|
| rustfs   | rustfs/rustfs:latest      | 9000, 9001   | S3-compatible object store             |
| redis    | redis:7-alpine            | 6379         | Cache / queues                         |
| neo4j    | neo4j:5-community         | 7474, 7687   | Graph DB (optional, `--profile graph`) |

## Quick start

```sh
cp .env.example .env
make up        # start postgres, rustfs, redis
make db-up     # apply all migrations
```

Optional graph DB:

```sh
docker compose --profile graph up -d neo4j
```

Local Postgres URL: `postgres://drip:drip@localhost:5432/drip`

## Source-of-truth

This repo owns:

1. **Migrations** — every dbmate migration lives in `db/migrations/`.
2. **Schema dump** — `db/schema.sql` is regenerated by `make db-dump`
   after every migration. Backend's sqlc reads this file to generate
   `internal/repository/*.go`.
3. **Seeds** — `db/seeds/dev.sql`.

The services in `../agentic` and `../backend` no longer own migrations
or schema. They connect to the Postgres started here.

## Service responsibilities

- **`../agentic`** (Python/FastAPI, iOS-facing) connects via
  `postgresql+asyncpg://drip:drip@localhost:5432/drip`. Reads/writes
  whatever tables it needs at runtime.
- **`../backend`** (Go/Fiber) connects via
  `postgresql://drip:drip@localhost:5432/drip?sslmode=disable`. Uses
  sqlc to generate type-safe Go from `db/schema.sql` in this repo.

Both services share the same database. Neither owns the schema.

## Adding a migration

```sh
make db-new name=add_widgets        # creates db/migrations/<ts>_add_widgets.sql
$EDITOR db/migrations/<ts>_add_widgets.sql
make db-up                          # applies it
make db-dump                        # refreshes db/schema.sql
```

Commit the new migration **and** the regenerated `db/schema.sql`. If the
change touches a table that backend reads, run `make sqlc` from
`../backend` to regenerate Go types.

## Common targets

| Target          | Effect                                                  |
|-----------------|---------------------------------------------------------|
| `make up`       | Start postgres + rustfs + redis                         |
| `make down`     | Stop containers (data preserved)                        |
| `make nuke`     | Stop + remove volumes (destroys local data)             |
| `make db-up`    | Apply pending migrations                                |
| `make db-down`  | Roll back the latest migration                          |
| `make db-status`| Show applied/pending migrations                         |
| `make db-new`   | Create a new migration file                             |
| `make db-dump`  | Regenerate `db/schema.sql` from the live DB             |
| `make seed-dev` | Load dev seed data                                      |
| `make seed-reset`| Drop, re-apply migrations, then seed                   |
