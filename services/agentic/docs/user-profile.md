## GET /api/v1/users/me

Returns the authenticated user's full profile including style preferences, sizing, connected integrations, and timestamps.

## PATCH /api/v1/users/me

Partially updates the user's profile. **All fields are optional** â€” only the fields you supply are written. Omitted fields are left exactly as they are. Safe to call multiple times.

**Auth:** `Bearer <access_token>`

**Content-Type:** `application/json`

**curl example**
```bash
curl -X PATCH http://localhost:8000/api/v1/users/me \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "display_name": "Alex",
    "location": "Paris, France",
    "style_identity": "Minimal with a bold accent",
    "preferred_styles": ["minimal", "old-money"],
    "preferred_colors": ["neutrals", "earth-tones"],
    "preferred_stores": ["cos", "uniqlo"],
    "tops_size": "tops-l",
    "bottoms_size": "bottoms-30",
    "shoes_size": "shoes-43",
    "outerwear_size": "outerwear-l",
    "budget_min": 50,
    "budget_max": 300,
    "mood": "calm"
  }'
```

### Request body

| Field | Type | Source | Description |
|---|---|---|---|
| `display_name` | `string \| null` | â€” | Public display name shown in the UI. |
| `location` | `string \| null` | â€” | City or region â€” used for weather context in outfit suggestions. |
| `style_identity` | `string \| null` | â€” | Free-text style self-description, injected into outfit AI prompts. |
| `preferred_styles` | `string[]` | `GET /onboarding/vibes` | Replaces the full list of vibe slugs. Pass all selected slugs, not just new ones. |
| `preferred_colors` | `string[]` | `GET /onboarding/colors` | Replaces the full list of color palette slugs. |
| `preferred_stores` | `string[]` | `GET /onboarding/stores` | Replaces the full list of store slugs. |
| `tops_size` | `string \| null` | `GET /onboarding/sizes` | Size slug from the `tops` group, e.g. `"tops-m"`. |
| `bottoms_size` | `string \| null` | `GET /onboarding/sizes` | Size slug from the `bottoms` group, e.g. `"bottoms-32"`. |
| `shoes_size` | `string \| null` | `GET /onboarding/sizes` | Size slug from the `shoes` group, e.g. `"shoes-42"`. |
| `outerwear_size` | `string \| null` | `GET /onboarding/sizes` | Size slug from the `outerwear` group, e.g. `"outerwear-l"`. |
| `budget_min` | `integer \| null` | â€” | Minimum budget in USD (â‰Ą 0). |
| `budget_max` | `integer \| null` | â€” | Maximum budget in USD (â‰Ą 0). |
| `mood` | `string \| null` | â€” | Current mood. One of: `happy` Â· `sad` Â· `energetic` Â· `calm` Â· `melancholic` Â· `angry` Â· `relaxed` Â· `focused` Â· `unknown`. |

> **Note on list fields (`preferred_styles`, `preferred_colors`, `preferred_stores`):** these replace the entire stored list. If the user previously had `["zara", "asos"]` and you send `["cos"]`, the result is `["cos"]`. To add a store, fetch the current list from `GET /users/me` and merge it client-side before sending.

### Response â€” `200 OK`

Returns the full updated `UserMeOut` object (same shape as `GET /users/me`).

**Error responses**

| Status | Cause |
|---|---|
| `401` | Missing or invalid JWT |
| `404` | User record not found |
| `422` | Invalid field value (e.g. unknown mood enum value) |

## POST /api/v1/users/me/avatar

Uploads or replaces the user's profile photo. The photo is used as the base image for AI outfit try-on previews generated by the shuffle prefetch worker.

See [`avatar-upload-endpoint.md`](./avatar-upload-endpoint.md) for the full spec.

## How profile fields flow downstream

| Field | Where it's used |
|---|---|
| `display_name` | Shown in the UI, social features |
| `location` | Weather lookup for outfit context |
| `style_identity` | Injected into outfit compose AI prompts |
| `preferred_styles` | Shuffle scoring â€” vibe alignment boosts candidate score |
| `preferred_colors` | Future: filter/rank items and suggestions by palette |
| `preferred_stores` | Future: prioritise scraper results from selected retailers |
| `tops_size` / `bottoms_size` / `shoes_size` / `outerwear_size` | Returned in profile; future: filter store products by size |
| `budget_min` / `budget_max` | Returned in profile; future: filter store products by price range |
| `mood` | Influences song suggestion matching in shuffle |
| `avatar_url` | Base image for AI try-on via shuffle prefetch |
